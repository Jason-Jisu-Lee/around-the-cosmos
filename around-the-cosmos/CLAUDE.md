# Around the Cosmos

Celestial idle/incremental game (the protagonist/center object is named **Maw**).
Pure vanilla JS + Canvas. No build step, no framework, no dependencies.

## What this game is
- The **Maw** (small dark center) **auto-generates stardust (✦) every second** once you buy **Cosmic Pulse**, and is orbited by **orbiters** that pay stardust each orbit
- **This is an idle game, NOT a clicker** - clicking never harvests. A click only **catches a passing comet** (the one interactive moment) or opens a body's info card
- You **start with 10 ✦** - just enough to buy the first Cosmic Pulse, which begins the income
- Tone: calm, meditative, slightly melancholy - "maw" means a gap or void

> **Current scope (`refine/v.3` - minimal base, rebuilding step by step):** The game
> starts with **10 ✦ and 0 orbiters** - the **Maw's pulse** is the core income; buy your first orbiter from there.
> **Economy note:** effects are **additive (fixed +amount/level), not doubling** - this keeps
> the scale from exploding. Multipliers are used sparingly (only Speed and Composition).
> Upgrade sections are now **per orbiter** (future tabs): **MAIN** / **DUST PARTICLES** / **ASTEROID** / **COMETS**.
> **MAIN:** **Cosmic Pulse** (8 levels, costs [10,50,120,250,580,1000,1850,3000], **+5 ✦/sec per level** - Maw auto-generates every `PULSE_INTERVAL`=1s once owned),
> then **Pulse Surge** (was Star Grasp; after 6th Cosmic Pulse; 5 levels, costs [1200,2200,3600,6300,10000], **+10 ✦ per pulse per level**),
> then **Deep Breath** (unlocks at `G.runDust ≥ 10000`; 8 levels, costs [2500,6000,13000,27000,55000,110000,220000,430000] - **every Nth pulse pays ×`deepBreathMult()`** (base ×2), with N = `deepBreathInterval()` = `13 − lvl` so lvl 1 → every 12th breath down to lvl 8 → every 5th; the multiplier is applied in the `simulation.js` pulse loop, NOT in `pulseValue()`; a Deep Breath also bounces harder (`deepBreath` click-FX, amp 0.26 vs 0.15) and flares Maw glow gold (`deepBreathFlash` in render.js) with a deeper tick `sfxDeepBreath`),
> then **Abyssal Breath** (unlocks at the same `G.runDust ≥ 10000`; **15 levels**, costs [3000,5000,8000,13000,20000,31000,49000,77000,120000,185000,290000,450000,700000,1100000,1700000] - **empowers Deep Breath, +0.2× to its multiplier per level** via `deepBreathMult()` = `2 + 0.2×lvl('abyssal')`, so Deep Breaths run **×2 → ×5** at max),
> then **Gravitational Pull** (an **advanced** upgrade - unlocks at `G.runDust ≥ 50000`, 50k stardust collected this universe; 5 levels, costs [12000,35000,120000,475000,1850000], each level adds
> **+1% of total orbiter payout to every pulse**), and **Resonance** (see below). Pulse value =
> `pulseValue()` = (5×Cosmic Pulse + 10×Pulse Surge + 0.01×gravpull×(total orbiter payout)) × denserCoreMult, rounded - **then doubled on a Deep Breath** in the pulse loop.
> The orbiter-payout sum (`orbiterPayoutSum`) uses each orbiter's **phase-averaged** payout (`avgPayout`, e.g. the moon) so the pulse doesn't flicker with the lunar phase.
> **(The old per-click Star Touch/Star Grasp and the separate Pulse auto-clicker upgrade are gone - the pulse IS the game.)**
> **DUST PARTICLES:** **Dust Particle** (after 2nd Cosmic Pulse; **one-time buy** that creates the first particle,
> cost 100, **+10 base payout**), then **Dust Particle Count** (after the first; **+1 particle/level**, 4 levels →
> 5 total, costs [600,1500,4000,9200] - all share one orbit on ring 0), **Dust Particle Payout** (**+10 to each dust
> particle's payout per level**, 5 levels, costs [200,800,2500,5600,14500]) and **Dust Particle Speed**
> (effective 82%→164%, costs [250,1000,3000,7500,16000]). (Like the other orbiters, the count/payout/speed
> upgrades populate only **once the first Dust Particle is bought**.)
> **ASTEROID:** **Asteroid** (after 5k stardust collected this universe, `G.runDust ≥ 5000`; a **single body**, one-time cost 2000, **+50 base payout**,
> own clump on the wider ring 1, with drifting dust motes) with **Asteroid Payout** (**+50/level**, 5
> levels, costs [2500,4500,10500,24000,58000]), **Asteroid Speed** (costs [3000,6000,13000,28000,62000]),
> and its unique **Asteroid Composition** (5 tiers Rock→Iron→Gold→Ice→Crystal→Diamond, recolors it + payout ×[1,1.2,1.4,1.6,1.8,2.0] (+0.2 per reforge, base ×1 → ×2.0 at max),
> costs [3500,11000,26000,60000,135000]). **MOON:** **Moon** (after 32k stardust collected this universe, `G.runDust ≥ 32000`; a **single body**, one-time
> cost 10000, **+200 base payout**, own clump on the widest ring 2 - pale, round, the slowest orbit) with
> **Moon Payout** (**+200/level**, 5 levels, costs [15000,50000,150000,330000,600000]), **Moon Speed**
> (the upgrade runs 100%→200%, scaled by a **0.663 base factor** (slowed 15% from 0.78) → 66.3%→132.6%, costs [9000,18000,35000,60000,100000]),
> and its unique **Lunar Phases** (`moonphase`, 5 levels, costs [12000,28000,55000,90000,175000]). Payout swings with the
> phase by **default**: ×0.75 at new moon up to ×1.25 at full moon; Lunar Phases shifts that range up by ×0.10 on **both**
> ends per level (lvl 5 → ×1.25..×1.75). Visible waxing/waning on a `MOON_CYCLE`=16s loop.
> **Resonance** (in MAIN, unlocks at `G.runDust ≥ 50000` - the same gate as Gravitational Pull, so they appear together; 5 levels) is a **global ×payout on every orbiter**,
> +0.10/level (×1→×1.5), and lights Maw's faint glow.
> **Only dust particles use a count upgrade**; all other orbiters are single bodies. **COMETS:** Comet Charm exists but is **disabled** (`unlock:()=>false`);
> comets still pay windfalls. Prestige, remnants, evolution, etc. are all out.
> Earlier ideas removed: New Planet, per-planet Orbit Payout/Speed, the per-planet tab UI
> (will return once there are >5 orbiters), First Light.

`SECTION_ORDER` = `['MAIN','DUST PARTICLES','ASTEROID','MOON','DWARF PLANET','COMETS']` - **one section per orbiter** (each becomes a tab eventually); MAIN holds the pulse upgrades (Cosmic Pulse, Pulse Surge) + Gravitational Pull + Resonance. `buildPanels` renders each section as a
**multi-open accordion** (`.acc`): all sections open by default, each independently
collapsible (state in `sectionOpen`). A section with no shown cards is omitted.

> **Orbiters note:** dust particles share **one clump on ring 0** (the only orbiter with a
> **count** upgrade - `Dust Particle` creates the 1st, `Dust Particle Count` adds 4 more → 5 total);
> the asteroid is a **single body** on its own clump on ring 1
> (wider/slower, `PLANET_DEF[1]`); the moon is a **single body** on its own clump on the widest ring 2
> (`PLANET_DEF[2]`, slowest). All future orbiters are single bodies too. Each type has its
> own Payout (+amount/lvl) and Speed upgrade. The asteroid's unique upgrade is **Composition** (`astcomp`); the moon's is **Lunar Phases** (`moonphase`), which widens its default phase payout range.
>
> **Component architecture:** each orbiter is a self-contained component in `orbiters/` that calls
> `registerOrbiter(...)`. `simulation.js` (tick / orbit payout / **buy reconciles body count to `count()`** + float label), `render.js`
> (orbit rings + clumps), `ui.js` (combined All-Orbiters-Payout), `state.js` `loadGame` (rebuild bodies
> from levels), and the cosmic info cards all **iterate `ORBITERS`** - so adding an orbiter means adding
> a file in `orbiters/` (+ its upgrades in config.js + a body array/clump in state.js), not editing the
> engine. Each component exposes `list/clump/clumpPos/ring/hoverR/color/pebbleR/payout/speed/make/count/
> bodyUpgrade/rows/labels` (+ optional `avgPayout` for a phase-averaged display value, and `round` to
> draw as a phased disc - see `orbiters/registry.js` header). On any buy, `buyUpgrade` syncs the orbiter's
> body array to `count()` (push/pop), so both the "create first body" upgrade and a "+1 count" upgrade add bodies.

**Hide completed:** maxed upgrades are **hidden by default** - but only **once the Moon has ever been owned**
(`G.moonEverOwned`, a **persistent milestone** set in `buyUpgrade` when `moon` is bought; saved + kept across
Accretion resets). **Before that** the toggle is hidden (`hud.js` toggles `#upg-controls` display on `moonEverOwned`)
and completed cards are **never hidden** (`isShown` returns maxed when `!G.moonEverOwned`). The "Hide completed" toggle
(`#show-completed`, pinned **bottom-right** of the upgrade panel) is **UNCHECKED by default** - completed upgrades **show by default**; hiding is opt-in.
`showCompleted` defaults **true**; the change handler sets `showCompleted = !checkbox.checked` (checking "Hide completed" hides the greyed maxed cards).
`isShown(u)` returns maxed cards when `showCompleted || !G.moonEverOwned`;
the fingerprint `visibleSig()` includes max-state + the toggle + `moonEverOwned` so the panel rebuilds when an upgrade
maxes, the toggle flips, or the Moon is first bought. **Newly-unlocked cards** get a one-time **yellow double-flash** highlight
(`.upg-new`, CSS `@keyframes upg-flash`, ~0.9s, a yellow overlay that fades in/out twice then clears; an
`animationend` listener removes `.upg-new` so it returns to its default look): `seenUpg` tracks ids already
rendered; `firstPanelBuild` suppresses the flash for everything visible on the initial build (only mid-play unlocks animate).

## Save system
localStorage key: `around_the_cosmos_v1` (migrates an old `lacuna_v1` save on load). Saves every 20s and on tab close. Settings: `around_the_cosmos_settings_v2` (volume scale rebased - see below; old `v1`/`lacuna_settings_v1` saves are **not** migrated so everyone lands on the new 75% default).

**Volume scale:** the music/sfx sliders run 0–100% but **75% is the reference loudness** (what used to be 100%); the extra range to 100% gives ~33% more headroom. `setMusicVolume`/`setSfxVolume` (sound/core.js) divide by **75**, not 100 (reference gains `0.5712*pct/75` music, `0.8262*pct/75` sfx). Defaults are **75%** (HTML slider value, `loadSettings`, the `_savedVols` boot fallback in main.js). **The master gain is 1.0** (raised from 0.75, +33% across music AND sfx - 75% was too quiet; `audioBoot` + the unmute value in `toggleMute` both carry it, and the brick-wall limiter still prevents clipping).

## Progressive upgrade unlock
- Each upgrade in `config.js` has an `unlock: () => boolean`; the card shows only when it returns true.
- `ui.js` computes a `visibleSig()` fingerprint each frame; when it changes (a new upgrade unlocks), the panel rebuilds automatically - so newly-unlocked cards appear without manual `buildPanels()` calls.
- To gate an upgrade behind a milestone, set its `unlock` fn, e.g. `unlock: () => lvl('touch') >= 2`.
- **Buying an upgrade only rebuilds the panel if `visibleSig()` changed** (else `updateCards()` refreshes values in place). A full rebuild recreates every card element and would kill an in-progress "new upgrade" highlight on a sibling - so card-value changes (e.g. leveling Cosmic Pulse) must NOT rebuild. `resetPanelAnimations()` (panels.js) clears `seenUpg` on a universe reset (debug Reset, future prestige) so re-unlocked cards flash again - needed because no page reload happens.
