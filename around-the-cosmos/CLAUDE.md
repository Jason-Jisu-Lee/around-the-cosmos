# Around the Cosmos

Celestial idle/incremental game (the protagonist/center object is named **the Lacuna**).
Pure vanilla JS + Canvas. No build step, no framework, no dependencies.

## What this game is
- The **Lacuna** (small dark center) **auto-generates stardust (✦) every second** once you buy **Cosmic Pulse**, and is orbited by **orbiters** that pay stardust each orbit
- **This is an idle game, NOT a clicker** — clicking never harvests. A click only **catches a passing comet** (the one interactive moment) or opens a body's info card
- You **start with 10 ✦** — just enough to buy the first Cosmic Pulse, which begins the income
- Tone: calm, meditative, slightly melancholy — "lacuna" means a gap or void

> **Current scope (`refine/v.3` — minimal base, rebuilding step by step):** The game
> starts with **10 ✦ and 0 orbiters** — the **Lacuna's pulse** is the core income; buy your first orbiter from there.
> **Economy note:** effects are **additive (fixed +amount/level), not doubling** — this keeps
> the scale from exploding. Multipliers are used sparingly (only Speed and Composition).
> Upgrade sections are now **per orbiter** (future tabs): **MAIN** / **DUST PARTICLES** / **ASTEROID** / **COMETS**.
> **MAIN:** **Cosmic Pulse** (8 levels, costs [10,50,150,250,400,600,800,1000], **+5 ✦/sec per level** — the Lacuna auto-generates every `PULSE_INTERVAL`=1s once owned),
> then **Pulse Surge** (was Star Grasp; after 6th Cosmic Pulse; 3 levels, costs [500,1000,1500], **+10 ✦ per pulse per level**),
> then **Gravitational Pull** (after Pulse Surge maxed; 2 levels, costs [5000,12000], each level adds
> **+1% of total orbiter payout to every pulse**), and **Resonance** (see below). Pulse value =
> `pulseValue()` = (5×Cosmic Pulse + 10×Pulse Surge + 0.01×gravpull×(total orbiter payout)) × denserCoreMult, rounded.
> The orbiter-payout sum (`orbiterPayoutSum`) uses each orbiter's **phase-averaged** payout (`avgPayout`, e.g. the moon) so the pulse doesn't flicker with the lunar phase.
> **(The old per-click Star Touch/Star Grasp and the separate Pulse auto-clicker upgrade are gone — the pulse IS the game.)**
> **DUST PARTICLES:** **Dust Particle** (after 2nd Cosmic Pulse; **one-time buy** that creates the first particle,
> cost 100, **+10 base payout**), then **Dust Particle Count** (after the first; **+1 particle/level**, 4 levels →
> 5 total, costs [500,1200,2500,4000] — all share one orbit on ring 0), **Dust Particle Payout** (**+10 to each dust
> particle's payout per level**, 5 levels, costs [150,500,1200,2000,3000]) and **Dust Particle Speed**
> (the upgrade runs 100%→200%, costs [200,600,1500,2500,4200]). (Like the other orbiters, the count/payout/speed
> upgrades populate only **once the first Dust Particle is bought**.)
> **ASTEROID:** **Asteroid** (after 2nd dust particle; a **single body**, one-time cost 1500, **+50 base payout**,
> own clump on the wider ring 1, with drifting dust motes) with **Asteroid Payout** (**+50/level**, 5
> levels, costs [1500,4500,10000,20000,36000]), **Asteroid Speed** (costs [2000,4500,9000,17000,30000]),
> and its unique **Asteroid Composition** (3 tiers Rock→Iron→Gold→Ice, recolors it + payout ×[1,1.25,1.5,1.75],
> costs [3000,8000,18000]). **MOON:** **Moon** (after you own the asteroid; a **single body**, one-time
> cost 8000, **+200 base payout**, own clump on the widest ring 2 — pale, round, the slowest orbit) with
> **Moon Payout** (**+200/level**, 5 levels, costs [8000,18000,36000,70000,120000]), **Moon Speed**
> (the upgrade runs 100%→200%, scaled by a **0.663 base factor** (slowed 15% from 0.78) → 66.3%→132.6%, costs [9000,18000,35000,60000,100000]),
> and its unique **Lunar Phases** (`moonphase`, 5 levels, costs [12000,28000,55000,90000,140000]). Payout swings with the
> phase by **default**: ×0.75 at new moon up to ×1.25 at full moon; Lunar Phases shifts that range up by ×0.10 on **both**
> ends per level (lvl 5 → ×1.25..×1.75). Visible waxing/waning on a `MOON_CYCLE`=16s loop.
> **Resonance** (in MAIN, unlocks after Pulse Surge is maxed — appears alongside
> Gravitational Pull; 5 levels, costs [3000,6500,12000,20000,32000]) is a **global ×payout on every orbiter**,
> +0.10/level (×1→×1.5), and lights the Lacuna's faint glow.
> **Only dust particles use a count upgrade**; all other orbiters are single bodies. **COMETS:** Comet Charm exists but is **disabled** (`unlock:()=>false`);
> comets still pay windfalls. Prestige, remnants, evolution, etc. are all out.
> Earlier ideas removed: New Planet, per-planet Orbit Payout/Speed, the per-planet tab UI
> (will return once there are >5 orbiters), First Light.

`SECTION_ORDER` = `['MAIN','DUST PARTICLES','ASTEROID','MOON','COMETS']` — **one section per orbiter** (each becomes a tab eventually); MAIN holds the pulse upgrades (Cosmic Pulse, Pulse Surge) + Gravitational Pull + Resonance. `buildPanels` renders each section as a
**multi-open accordion** (`.acc`): all sections open by default, each independently
collapsible (state in `sectionOpen`). A section with no shown cards is omitted.

> **Orbiters note:** dust particles share **one clump on ring 0** (the only orbiter with a
> **count** upgrade — `Dust Particle` creates the 1st, `Dust Particle Count` adds 4 more → 5 total);
> the asteroid is a **single body** on its own clump on ring 1
> (wider/slower, `PLANET_DEF[1]`); the moon is a **single body** on its own clump on the widest ring 2
> (`PLANET_DEF[2]`, slowest). All future orbiters are single bodies too. Each type has its
> own Payout (+amount/lvl) and Speed upgrade. The asteroid's unique upgrade is **Composition** (`astcomp`); the moon's is **Lunar Phases** (`moonphase`), which widens its default phase payout range.
>
> **Component architecture:** each orbiter is a self-contained component in `orbiters/` that calls
> `registerOrbiter(...)`. `simulation.js` (tick / orbit payout / **buy reconciles body count to `count()`** + float label), `render.js`
> (orbit rings + clumps), `ui.js` (combined All-Orbiters-Payout), `state.js` `loadGame` (rebuild bodies
> from levels), and the cosmic info cards all **iterate `ORBITERS`** — so adding an orbiter means adding
> a file in `orbiters/` (+ its upgrades in config.js + a body array/clump in state.js), not editing the
> engine. Each component exposes `list/clump/clumpPos/ring/hoverR/color/pebbleR/payout/speed/make/count/
> bodyUpgrade/rows/labels` (+ optional `avgPayout` for a phase-averaged display value, and `round` to
> draw as a phased disc — see `orbiters/registry.js` header). On any buy, `buyUpgrade` syncs the orbiter's
> body array to `count()` (push/pop), so both the "create first body" upgrade and a "+1 count" upgrade add bodies.

**Hide completed:** maxed upgrades are **hidden by default** — but only **once the Moon has ever been owned**
(`G.moonEverOwned`, a **persistent milestone** set in `buyUpgrade` when `moon` is bought; saved + kept across
Accretion resets). **Before that** the toggle is hidden (`hud.js` toggles `#upg-controls` display on `moonEverOwned`)
and completed cards are **never hidden** (`isShown` returns maxed when `!G.moonEverOwned`). The "Hide completed" toggle
(`#show-completed`, pinned **bottom-right** of the upgrade panel) is **checked by default**.
`showCompleted` defaults **false**; the change handler sets `showCompleted = !checkbox.checked` (unchecking shows the greyed maxed cards).
`isShown(u)` returns maxed cards when `showCompleted || !G.moonEverOwned`;
the fingerprint `visibleSig()` includes max-state + the toggle + `moonEverOwned` so the panel rebuilds when an upgrade
maxes, the toggle flips, or the Moon is first bought. **Newly-unlocked cards** get a one-time **yellow double-flash** highlight
(`.upg-new`, CSS `@keyframes upg-flash`, ~0.9s, a yellow overlay that fades in/out twice then clears; an
`animationend` listener removes `.upg-new` so it returns to its default look): `seenUpg` tracks ids already
rendered; `firstPanelBuild` suppresses the flash for everything visible on the initial build (only mid-play unlocks animate).

Other mechanics:
- **Orbiters / payout**: three clumps (dust ring 0, asteroid ring 1, moon ring 2), rendered by the shared `drawClump()` helper (render.js).
  - **Dust** — `G.planets[]` (count = `dust`(0/1) + `dustcount`, capped 5), each `{localPhase,localR,localSpin,pulse,shape}`, travel as a clump on `G.clump{angle,nextTop}` (ring 0, period `PLANET_DEF[0].period`=6s). Speed = `(2π/period)×dustSpeed()`, `dustSpeed()`=`0.82 × dustspd.mult(lvl)` (mult=1+0.2×lvl) — a **0.82 base factor** (base speed reduced 18%) so effective speed runs **82% → 164%**. On top-cross each pays `orbiterPayout()` = `round((10 + 10·lvl(dustpay)) × resonanceMult())` (additive base × global Resonance; in orbiters/dust.js). Small grey pebbles (radius `PLANET_DEF[0].radius/3+2`), local orbit 5–12px.
  - **Asteroid** (single body) — `G.asteroids[]` (a **fixed** `ASTEROID_SHAPE` outline — rounded/lumpy, not the old per-load random spiky shape — + a `motes` array), clump on `G.asteroidClump` (ring 1, period `PLANET_DEF[1].period`=9.5s). Speed = `asteroidSpeed()`=`0.88 × astspd.mult(lvl)` (**0.88 base factor**, base reduced 12% → effective 88% → 176%). On top-cross pays `asteroidPayout()` = `round((50 + 50·lvl(astpay)) × ASTEROID_COMP.mult[lvl(astcomp)] × resonanceMult())` (rounded → integer; in orbiters/asteroid.js). Bigger pebble, radius `(PLANET_DEF[1].radius/3+4)×1.95`, local orbit 8–16px; **color = `asteroidColor()` = the current Composition tier** (`ASTEROID_COMP.colors`, Rock keeps `#7a6a55`). Carries 6 decorative **motes** — tiny specks that drift around it at all times (drawn in `drawClump` when `o.motes` exists; much smaller than a dust orbiter). Also gets **soft 2D shading for 3D roundness** (gated on `o.motes`): after the body is filled, the path is clipped and a **radial gradient** darkens the far side (transparent at the upper-left light, up to `rgba(20,16,12,0.34)` at the far edge). It's drawn in **world orientation** (`ctx.rotate(-la*1.5)` undoes the body spin) so the light stays fixed at upper-left while the rock tumbles. The asteroid **shape is unchanged** — it's just an overlay. **Dust pebbles get the same far-side shadow but much fainter** (the `else` branch in `drawClump`: same upper-left light + clip + world-orientation, gradient up to only `rgba(20,16,12,0.20)` vs the asteroid's 0.34) — just a touch of roundness.
  - **Moon** (single body) — `G.moons[]`, clump on `G.moonClump` (ring 2, period `PLANET_DEF[2].period`=13s — the slowest). **Sits exactly on the orbit line** (`localR:0` — no local wobble, unlike dust/asteroid). Speed = `moonSpeed()`=`0.663 × moonspd.mult(lvl)` (**0.663 base factor**, slowed 15% from 0.78 → effective 66.3% → 132.6%). On top-cross pays `moonPayout()` = `round((200 + 200·lvl(moonpay)) × moonPhaseMult() × resonanceMult())` (rounded → integer; in orbiters/moon.js). ## Accretion (prestige) — flow + animation + Mass upgrade tree
**The flow** (`ui/accretion.js`): **`#accretion-btn`** (bottom-left, Dusty-Blue accent) is shown only when **`G.runDust >= ACCRETION_THRESHOLD && massGain() >= 1 && !accreting`** (`hud.js`) — gated on **`runDust`** (the *current-universe* stardust, the same number shown as "Total Stardust Collected"), **not** lifetime `totalDust` — so the button matches the displayed figure and appears exactly when the run hits 200k (the earlier lifetime gate made it show "early" after a save/accretion left `totalDust` already high). Toggled via explicit `'block'` — `''` falls back to CSS `display:none`. Pressing it opens **`#acc-confirm`** directly (a top-level fixed modal, **themed Nunito/parchment** — not an OS warning box; choices are **Stay** / **Accrete**) — **not** the upgrade page. `accConfirmBody()` is rebuilt each open, so it appends a **music-muted hint** (`.acc-mutehint`, "♪ … recommend unmuting") **only when the music specifically is silenced** — `musicMuted()` = `SoundSystem.isMuted()` (master mute) **or** the `#vol-music` slider at 0; effects-only being muted does NOT trigger it (the collapse animation is music-timed). Confirm → `startAccretionSequence()`:
1. `commitAccretion()` — **credits Mass immediately** (the major event happens at confirm, not later).
2. sets **`accreting=true`** (main.js loop skips `tickWithDebug` → sim/earning/comets frozen; **stays true through the whole Mass page**, only cleared on Return), clears the comet, dismisses any pinned info card, and **stops the music** (`SoundSystem.stopMusic()`; SFX are already silent since the sim is frozen), then plays the one-shot cue **`sound/music/accretion-1.mp3`** once (`accretionAudio`, an HTML5 `Audio`, vol 0.68, gated on `!SoundSystem.isMuted()`; stopped on Return).
3. `runAccretionFx()` plays a **Singularity collapse & Rebirth** on the full-window `#accretion-fx` canvas (z110, **opaque** — covers the game; centered on the on-screen Lacuna). It's **choreographed to `accretion-1.mp3` as 5 timed beats** (`FX_TL` = windup 15.0 / absorb 4.15 / collapse 2.55 / converge 10.0 / rebirth 2.5 s; ~34.3s total): **WINDUP** — orbiters (snapshotted as polar bodies via `o.clumpPos()`/`o.color()`, each seeded with its **real current angular speed** `(2π/PLANET_DEF[ring].period)×o.speed()`) spin up from their current speed to `SPIN_TOP`=14× on the bright parchment; **ABSORB** — they spiral into the Lacuna while the background darkens (the starfield is hidden on the bright bg, alpha purely `dark`-driven, and fades in only here); **COLLAPSE** — white flash + gold shockwave (the flash lands at **19.15s**); **CONVERGE** — a 150-star field accelerates inward and **recycles** any swallowed star to the rim (endless universe) for the first ~6s, then over the last 4s the inflow **decelerates to a stop in place** (no outward push) and settles to a calm field, stars on the Lacuna fading out; **REBIRTH** — purple bloom over the calm field. After rebirth it **holds the final frame for `FX_HOLD`=3s** (music still playing), then `onDone` runs `resetUniverse()`+`openAccretion()` and **fades the Mass page in over 3s** over the still-showing FX backdrop (the Mass page is z120 > FX z110), removing the FX canvas after the fade. The **accretion cue keeps playing** through the hold + fade and into the Mass page (only stopped on Return). Tunable timing lives in the throwaway lab `test/singularity-demo.html` (sliders per beat + Play+Music sync) — delete `test/` once locked.
4. on finish → `resetUniverse()` (full reset keeping `mass`/`massEarned`/`totalDust`/`gameTime`), then **`openAccretion()`** → the **Mass page** (`#accretion-screen`). It's **opaque parchment** (the game — Lacuna/orbiters/comets/panels/observatory/header — is fully hidden and stays frozen) with its own **`#acc-stars`** canvas running the **same looping stars** (`drawStars(t, g, w, h)`, now parametric in stars.js) so only "background colour + stars" show behind the UI. Tree is **top-aligned** (`.acc-stage justify-content:flex-start`) so it reads top-to-bottom without scrolling; big nodes (≈210px). The **bottom-center** button (`#acc-return`, larger, `--accretion` fill) plus a top-right **`#acc-undo`** ("↶ Undo", shown only in spend mode when `canUndoMass()`) are the only chrome — there is **no "Accretion" eyebrow/title** label. **Two modes** (`accBrowse` flag):
- **Post-Accretion mode** (this flow, `accBrowse=false`): opaque, game frozen, **Mass is spendable** (nodes clickable). `#acc-return` reads "Return to the Cosmos" and routes through `onAccBack` → `openAccLeaveConfirm()` (the **`#acc-leave-confirm`** modal warns that Mass can only be spent here, right after accreting — once you leave you can only *view* the tree until the next Accretion). Confirming → `closeAccretion()` (hides page, stops stars, `accreting=false`, restarts music → new universe resumes).
- **Browse mode** (`openMassBrowse()` from `#mass-upgrades-btn`, `accBrowse=true`): the **game keeps running** behind a **translucent** page (`.browse` class — `rgba(244,240,232,0.8)`, no `#acc-stars`; the real sky shows through), the tree is **view-only** (`accNodeHTML` drops the `data-id`/buy affordance when `accBrowse`), and the button reads **"Back"** → `closeMassBrowse()` (no confirm, no freeze, no music change).
Escape routes the same way (leave-confirm → acc-confirm → `onAccBack`). There is no in-page Accrete/× button.
- **Mass** (`G.mass`): `massEarnable()` = `floor(3 × √(totalDust / 200000))` (200k→3, 800k→6, 1.8M→9, 5M→15), all-time-stardust-based. **Grant math (forward-only Greater Collapse):** `baseMassGain()` = `massEarnable() − G.massEarned` (anti-farm, coefficient-3 base); `massGain()` = `round(baseMassGain() × greaterCollapseMult())` (what's shown + granted). `commitAccretion()` adds the **multiplied grant** to `G.mass` but only the **base** to `G.massEarned` — so buying Greater Collapse boosts future accretions without retroactively dumping Mass. `commitAccretion()`/`baseMassGain()`/`resetUniverse()` live in state.js.
- **Mass upgrade tree** (`upgrades/mass.js`, 8 real nodes, **few-but-powerful**): **Lacuna** — *Denser Core* (×pulse income, **+50%/lvl**, 5 lvls, **tier 0**) · *First Light* (**tier 1**, after Denser Core is maxed, alongside Event Horizon; start each universe with ✦ **1,000 → 5,000 → 20,000**, 3 lvls, costs [3,6,10]). **Orbiters** — *Heavier Bodies* (global ×orbiter payout, +50%/lvl, folded into `resonanceMult`, 5 lvls) · *Retained Companions* (start with Dust→Asteroid→Moon pre-formed, 3 lvls). **Phenomena** — *Brighter Tails* (×comet windfall, +50%/lvl, 4 lvls) · *Comet Shower* (gap ×0.85^lvl = sooner, 3 lvls). **Cycles** — *Greater Collapse* (×Mass/accretion = 1+lvl/3, 3 lvls) · *Lunar Favor* (×moon payout, +10%/lvl, 3 lvls). Effects hook the live economy via the accessors in mass.js: `pulseValue` (Denser Core), `resonanceMult` (Heavier Bodies → every orbiter + comet combined-payout + gravpull), `moonPayout`/`moonAvgPayout` (Lunar Favor), `catchComet`/observatory comet value (Brighter Tails), `randCometGap` (Comet Shower). **First Light + Retained Companions** apply at universe birth via `applyMassUniverseStart()` — called by `resetUniverse()` and re-called on every Mass-page buy (the universe is frozen there, so it's safe to re-derive starting ✦ + pre-owned bodies). ## Progressive upgrade unlock
- Each upgrade in `config.js` has an `unlock: () => boolean`; the card shows only when it returns true.
- `ui.js` computes a `visibleSig()` fingerprint each frame; when it changes (a new upgrade unlocks), the panel rebuilds automatically — so newly-unlocked cards appear without manual `buildPanels()` calls.
- To gate an upgrade behind a milestone, set its `unlock` fn, e.g. `unlock: () => lvl('touch') >= 2`.
- **Buying an upgrade only rebuilds the panel if `visibleSig()` changed** (else `updateCards()` refreshes values in place). A full rebuild recreates every card element and would kill an in-progress "new upgrade" highlight on a sibling — so card-value changes (e.g. leveling Cosmic Pulse) must NOT rebuild. `resetPanelAnimations()` (panels.js) clears `seenUpg` on a universe reset (debug Reset, future prestige) so re-unlocked cards flash again — needed because no page reload happens.
